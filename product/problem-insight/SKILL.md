# 产品问题洞察（第一性原理）

这个 skill 用于从用户问题出发，拆解本质需求，并推导出更合理的产品解法方向。

它专注于回答以下问题：

- 用户说的需求是什么
- 用户真实在经历的问题是什么
- 用户真正想完成的目标是什么
- 当前方案为什么没解决问题

它不负责产出完整 PRD，也不直接拆执行任务。若已经明确要定义需求，使用 `prd-breakdown`；若已经明确要排执行计划，使用 `task-breakdown`。

## 工作流程

1. 识别用户表达的问题，也就是表层需求。
2. 区分用户说了什么，与用户真实在经历什么。
3. 优先分析用户行为、场景和触发条件，而不是只分析观点。
4. 拆解问题的第一性原理：目标、本质和约束。
5. 判断当前方案的问题：成本、体验和心理阻力分别在哪里。
6. 推导更合理的解法方向，而不是直接堆具体功能。
7. 识别用户未表达但关键的机会点。

## 输出要求

输出应尽量包含：

- 表层需求
- 深层问题
- 用户目标
- 关键场景
- 当前方案问题
- 第一性原理拆解
- 解法方向
- 潜在机会点
- 已知事实
- 关键假设
- 待验证问题

## 约束

- 不直接根据用户建议设计功能。
- 优先分析行为和场景，而不是观点。
- 解法必须回到用户目标，而不是停留在功能层。
- 区分问题与解决方案，避免混淆。
- 明确标注哪些结论来自事实，哪些来自推断。
