# 产品执行拆解

这个 skill 用于把已经定义过的需求，继续拆成执行层可以直接使用的任务结构。

它不负责重新定义问题，也不替代完整 PRD。若需求本身还模糊，先使用 `problem-insight` 或 `prd-breakdown`。

## 工作流程

1. 先确认输入是否已经具备基本需求定义。
2. 找出主链路任务、前置依赖和可并行项。
3. 按阶段、模块或里程碑拆解执行路径。
4. 将每一项任务写成明确交付结果，而不是空泛动作。
5. 标出阻塞项、风险项和需要协作的角色。
6. 输出一个能直接进入跟进、排期和协作的任务清单。

## 输出要求

输出应尽量包含：

- 执行目标
- 拆解逻辑
- 任务列表
- 优先级
- 前置依赖
- 并行关系
- 负责人建议
- 里程碑
- 风险与阻塞

## 约束

- 任务颗粒度要能被实际跟进和验收。
- 不要只写“优化”“支持”“处理”这类空泛动作。
- 任务顺序要体现依赖关系，而不是随意罗列。
- 若输入需求不完整，要先指出缺口再拆解。
- 不要把需求分析和方案讨论重新混进执行清单里。
