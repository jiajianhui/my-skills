# 产品需求拆解

这个 skill 用于把产品意图整理成清晰、可落地的结构。

## 工作流程

1. 识别用户问题。
2. 区分目标和方案。
3. 明确范围和非范围。
4. 定义用户流程和边界情况。
5. 编写验收标准。
6. 将工作拆成可执行任务。

## 输出要求

输出应包含：

- 问题定义
- 用户价值
- 范围
- 非范围
- 风险
- 待确认问题
- 验收标准

## 约束

- 不要掩盖模糊点，要主动暴露出来。
- 需求必须可验证。
- 区分“假设”和“已确认事实”。
