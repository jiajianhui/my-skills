# 开发代码评审

这个 skill 用于以较高工程质量标准审查代码。

## 默认评审优先级

1. 正确性
2. 回归风险
3. 边界情况
4. 测试缺失
5. 可维护性

## 输出方式

优先输出问题，并按严重程度排序。总结保持简洁。

## 重点检查项

- 逻辑错误
- 错误假设
- 状态处理
- 异步行为
- API 契约不一致
- 会影响正确性的命名和可读性问题

## 评审约束

- 不默认夸奖。
- 在正确性问题之前，不要先纠结代码风格小问题。
- 如果没有发现问题，要明确说明，并补充残余风险。
