## 使用说明

写这类文档时，优先遵守下面几点：

- 优先讲输入和输出长什么样。
- 优先讲“这一步在做什么”，不要空话。
- 示例一定尽量贴近真实代码。
- 不要把整份代码逐行翻译。
- 如果只是解释一个函数，就缩短到该函数范围，不要硬展开成整份教程。
