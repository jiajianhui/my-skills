# 视觉原则

## 整体气质

目标气质应是“有把握的极简”，而不是空洞的简单：

- 结构清晰
- 噪音很低
- 排版精准
- 对比受控
- 克制而高级

设计应当让人感觉经过取舍和打磨，而不只是“元素比较少”。

## 布局

- 使用更大的外边距和更平静的区块节奏。
- 每个区块只讲一个主观点。
- 相比堆很多同权重卡片，更推荐少量区块但拉开对比。
- 打破单调应依靠尺度变化、画面裁切、强对齐，而不是随意装饰。

## 排版

- 标题要明确、利落、带判断。
- 正文要短、直白、高信息密度。
- 不要堆太多字号。少而清晰的层级更好。
- 如果“风格化”和“可读性”冲突，正文优先可读，标题可以更有性格。

## 色彩

- 先搭建稳固的中性色基础。
- 强调色只用于焦点、CTA、激活态和关键数据。
- 除非品牌本身明确支持，否则不要使用多强调色系统。
- 高级感往往更多来自明度和层级对比，而不是高饱和撞色。

## 表面与层次

- 优先选择平面或轻层叠的表面关系，不要依赖厚重阴影。
- 只有在边框确实提升清晰度时才使用。
- 卡片应该因为“有必要分组信息”而存在，而不是默认就加。
- 背景可以有轻微渐变、纹理或微光，但必须服务于层级，而不是抢戏。

## 动效

- 动效应当帮助建立结构和引导注意力。
- 更推荐错落出现、面板切换、轻微视差，而不是装饰性循环动画。
- 任何不能帮助理解和聚焦的动画，都应该删掉。

## 如何转译参考风格

从 Apple 借鉴：

- 平静的留白
- 高级的节奏感
- Hero 区表达直接清楚

从 Linear 借鉴：

- 严肃的产品感
- 有纪律的明暗对比逻辑
- 精准的产品界面细节

不要借鉴：

- 具体组件形状
- 具体渐变样式
- 具体文案节奏
- 具体页面构图
