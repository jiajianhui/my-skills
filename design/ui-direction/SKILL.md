# 设计方向

这个 skill 用于需要明确视觉方向的界面设计任务，而不是产出泛化、模板化的 UI。

## 默认方向

目标视觉语言应当具备以下特征：

- 简洁
- 现代
- 克制
- 高级
- 留白充足
- 以产品表达为中心

基础参考方向是以下两者的交集：

- Apple：层级平静清晰、留白大胆、克制而高级
- Linear：产品表达明确、信息清楚、动效精致、排版严肃利落

不要直接照搬任何品牌。提炼原则，再结合当前任务进行转译。

## 工作流程

1. 先确认设计对象：落地页、产品页、仪表盘、功能介绍、设置页、引导页，还是营销物料。
2. 再确认核心目标：传达价值、提升注册转化、增强信任感、减少混乱，还是提高聚焦度。
3. 在开始设计前，先用一句话定义本次视觉立场。
4. 明确基础系统：
   - 排版语气
   - 间距节奏
   - 色彩策略
   - 表面处理方式
   - 动效方式
5. 最终产出的方向必须有明确个性和判断，不要像常见 SaaS 模板的拼装结果。

## 输出要求

在提出设计方案时，至少包含：

- 一句简短的创意方向描述
- 一个清晰的布局概念
- 排版建议
- 色彩与材质建议
- 组件行为说明
- 明确指出应避免什么

如果用户要求进一步落地实现，需要把视觉方向翻译成具体的界面结构和样式指导。

## 设计约束

- 优先强化层级，而不是堆砌装饰。
- 优先通过留白和对齐建立秩序，而不是到处加边框。
- 颜色系统要收敛。通常一套中性色加一个克制的强调色就够了。
- 除非任务明确需要，否则不要做成轻飘、可爱、典型创业公司默认风格。
- 避免默认紫色渐变、夸张玻璃拟态、以及没有目的的“AI感”装饰。
- 界面必须有删减感。宁可删除弱内容，也不要为了填满版面而凑模块。

## 需要按需读取的参考

- 在定义整体风格和布局语言时，读取 `references/visual-principles.md`
- 在评审现有设计或检查输出质量时，读取 `references/review-checklist.md`
- 在需要复用设计提案结构时，使用 `assets/ui-brief-template.md`

## 关于 assets

`assets/` 用于存放可复用的产出模板。在这个 skill 里，放的是一个可直接套用的设计提案模板，适合落地页、首页或功能概念方案。
